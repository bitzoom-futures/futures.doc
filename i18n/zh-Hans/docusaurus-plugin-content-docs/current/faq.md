# 常见问题（FAQ）

## 通用问题

### Bitzoom 合约 API 可以做什么？

你可以通过 API：
- 获取实时行情
- 下单与撤单
- 查询仓位与余额
- 构建自动化交易策略

### 限频是多少？

| 接口类型 | 限频 |
|----------|------|
| 公共接口 | 10 次/秒 |
| 私有接口 | 5 次/秒 |
| 下单接口 | 10 次/秒 |
| WebSocket 连接 | 每个 IP 最多 5 条 |

### 是否有测试环境？

有。

| 环境 | Base URL |
|------|----------|
| 生产 | `http://119.8.50.236:8088` |
| 测试 | `https://testnet.bitzoom.com` |

## 鉴权

### Token 过期怎么办？

出现 `-1002 Unauthorized` 时，重新获取 Token 并更新服务配置。

### 可以一个 API Key 给多个应用使用吗？

可以，但建议按应用拆分 API Key，便于权限管理和审计。

## 交易

### 支持哪些订单类型？

`MARKET`、`LIMIT`、`STOP`、`TAKE_PROFIT`、`STOP_MARKET`、`TAKE_PROFIT_MARKET`。

### `timeInForce` 是什么？

| 值 | 含义 |
|----|------|
| `GTC` | 一直有效直到成交或撤单 |
| `IOC` | 立即成交，未成交部分取消 |
| `FOK` | 要么全部成交，要么全部取消 |
| `GTX` | 只挂单，若会立即成交则拒绝 |

## WebSocket

### 为什么连接会断开？

常见原因：
- 连接达到 24 小时上限
- 长时间无心跳
- 订阅或消息发送过快

### 订阅上限是多少？

- 单连接最多 200 个流
- 单 IP 最多 5 条连接
- 每秒最多 10 次订阅请求
